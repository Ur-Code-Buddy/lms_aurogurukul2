Aurogurukul is meant to cover the full loop of running a teaching business online: authoring, delivery, payments, site presence, and audience tools, without spreading that work across many disconnected products.

## Principles

1. **Straightforward operations** — Hosting, upgrades, and incident response should be predictable whether you use the managed environment or a deployment you run yourself.

2. **Clear surfaces** — Admin, learner, and public experiences should stay coherent as the product grows, so schools are not forced into one-off workarounds for routine tasks.

3. **Room to specialize** — Domains, integrations, and deployment layout need to adapt to real schools, including private networks and regulated contexts.

## How schools use it

| Model | Typical use |
| ----- | ----------- |
| **Managed (aurogurukul.app)** | Teams that want the application operated for them, with standard networking and updates. |
| **Dedicated hosting** | Teams that need custom infrastructure, networking, or data residency. Start with [Self hosting](/en/self-hosting/introduction). |
